## 0.0.0 — pre-release development

The package has not been published yet (current version `0.0.0`); it is being built
epic-by-epic. The milestones that shaped today's public API and behaviour:

- **Public component API (E7-S1):** the brief-§8 inputs (`template`, `data`, `config`,
  `theme`) and outputs (`rendered`, `pageChange`, `error`), fully typed.
- **Render pipeline (E7-S2):** one validate → bind → paginate → render pass over the
  shared engine + renderer; failures surface through `(error)`, never thrown.
- **Zoom & navigation (E7-S3 / E7-S4)** and **loading / empty / error states (E7-S5).**
- **Configurable toolbar (E8-S1):** per-button show/hide plus a host custom-action slot.
- **Print (E8-S2)**, **Export PDF** through a swappable `PdfExporter` (E8-S3), and
  **Watermark (E8-S4).**
- **Download source (E8-S5)**, **in-report Find (E8-S6)**, and an **optional thumbnail
  rail (E8-S7).**
- **Packaging & integration (E9):** Angular Package Format build with
  engine/renderer/schema bundled in (E9-S1); wide Angular peers `>=20` with
  tree-shake + SSR safety (E9-S2); real integration proof in `viewer-demo` (E9-S4);
  documented theming & style isolation (E9-S5); and these usage docs — README
  quick-start, TypeDoc API reference, Storybook stories and this changelog (E9-S6).
