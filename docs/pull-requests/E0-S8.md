## Summary

Establishes the **design-token contract** and the first real `ui-kit` component.
Tokens are plain `--rdr-*` **CSS custom properties** authored from
`docs/ui-mockups/.../design.md` (brief §12.3 applied: accent indigo `#4F46E5`,
quiet grayscale chrome), covering color, spacing (8px grid), typography, radii,
**elevation**, focus-ring and motion. A token-driven **`Button`** replaces the
E0-S2 `UiKit` placeholder, and a Storybook **Theme** toggle renders it (and a
token Foundations gallery) in **light and dark** — the story-specific QA. Full
guide: [`docs/design-system/tokens.md`](../design-system/tokens.md).

### Key decisions (see [ADR 0004](../adr/0004-design-tokens-theming.md))

- **Tokens = CSS custom properties** in
  [`libs/ui-kit/src/styles/tokens.css`](../../libs/ui-kit/src/styles/tokens.css),
  the single source of truth. Framework-neutral: backs scoped component CSS now,
  and can back a Tailwind `theme` in Epic 5 without changing the contract.
- **Light is authoritative (`:root`); dark is a provisional scaffold** under
  `.rdr-theme-dark`. `design.md` specifies light only, so the dark ramp is
  **derived** (slate surfaces, indigo accent lightened to `#818cf8` for AA) and
  flagged to be revisited once a dark spec exists. Class-based theming (not a
  media query) keeps it explicit and matches the future viewer CSS-var theming.
- **Tailwind deferred** (the story marks it *optional*). The designer is still an
  E0-S2 skeleton — real chrome is Epic 5, where Tailwind will be wired into
  `apps/designer` only, bound to these tokens. Keeping `ui-kit` token + scoped
  CSS ensures nothing the **viewer** can transitively reach depends on Tailwind
  (UI-kit-light rule, brief §3).
- **`Button` is an attribute selector** (`button[rdr-button]`) on a native
  `<button>`, so keyboard/focus/`type`/form/`:disabled` semantics come for free
  and the component only paints from tokens. This extends `ui-kit`'s
  `@angular-eslint/component-selector` to `type: ['element', 'attribute']` (same
  `rdr` prefix + kebab style) — the convention Angular Material uses for
  `button mat-button`; scoped to `ui-kit`'s ESLint config only.
- **Storybook loads tokens via the builder `styles` option**, not a `preview.ts`
  CSS `import` — the Angular Storybook (webpack5) builder has no CSS loader for
  preview-side imports, so global stylesheets go through `styles` (as in the app
  builder).

## Acceptance criteria

| Criterion | Evidence |
| --- | --- |
| CSS-custom-property token set (color, spacing, typography, radii, **elevation**) | `tokens.css` — all `--rdr-*`, grouped; documented in `tokens.md` |
| `ui-kit` scaffolding | `Button` (`button[rdr-button]`, primary/secondary/ghost) replaces the placeholder; barrel exports `Button` + `ButtonVariant` |
| Tailwind configured for `apps/designer` (**optional**) bound to tokens | **Deferred** to Epic 5 (ADR 0004) — designer instead consumes tokens directly as a global stylesheet |

  `nx build-storybook ui-kit` → *"Storybook build completed successfully"*; the
  built `index.json` lists `ui-kit/Button` (Primary/Secondary/Ghost/Disabled/
  Gallery + autodocs) and `ui-kit/Foundations` (Colors/Typography). A **Theme**
  toolbar (light/dark) wraps each story in a themed surface via the preview
  decorator. `nx storybook ui-kit --smokeTest --ci` → *"Smoke tests passed"*
  (zoneless, no Zone.js).

`nx build designer` (global `tokens.css` → `styles` bundle 2.89 kB, within
budget) · `prettier --check .`.
