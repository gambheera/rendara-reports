## Summary

Adds Playwright **pixel-diff snapshotting** with a deterministic, OS-stable
rendering environment, so rendering regressions are caught automatically (brief
§9: "no unreviewed pixel diffs"). This is the harness; the renderer it guards
lands in Epic 4. Full guide:
[`docs/testing/visual-regression.md`](../testing/visual-regression.md).

### Key decisions (see [ADR 0001](../adr/0001-visual-regression-determinism.md))

- **Baselines are Linux-canonical, CI-generated.** Screenshots are
  OS-rasterizer-specific (Windows DirectWrite ≠ Linux FreeType); local is
  Windows, CI is Linux. Only `*-linux.png` is committed; `*-win32.png` /
  `*-darwin.png` are git-ignored throwaways. **No baselines committed yet** —
  E0-S3 seeds them via `update-snapshots` (the chosen "harness now, baselines in
  CI later" approach; no Docker dependency).
- **Self-contained, deterministic fixtures.** The example spec renders via
  `page.setContent()` with an **embedded** vendored Inter woff2 (OFL) and awaits
  `document.fonts.ready`; fixed `1280×720` viewport + `deviceScaleFactor: 1`,
  forced light scheme, `toHaveScreenshot({ animations: 'disabled', caret: 'hide' })`
  + small diff tolerances. The only remaining variable is the OS rasterizer.
- **Separate `visual` target, not `e2e`.** The config is named
  `playwright.visual.config.mts` so the `@nx/playwright/plugin` does not infer an
  `e2e` target — visual regression is opt-in (`pnpm visual`) and never blocks the
  functional e2e gate while baselines are unseeded.
- Decoupled from app UI (which is still skeleton and will churn); real report
  snapshots are added against this harness once the renderer exists.

## Story-specific QA

An intentional visual change produces a reviewable failing diff: changing the
fixture rectangle colour (`#4f46e5` → `#dc2626`) made `pnpm visual` **fail** and
emit `harness-actual.png` + `harness-diff.png` (under the git-ignored
`dist/.playwright/…`); reverted. Locally the pipeline is proven end-to-end via a
throwaway `-win32` baseline (git-ignored); canonical `-linux` baselines arrive
with E0-S3.

Other gates stay green and independent: `nx run-many -t lint` (10) ·
`nx run-many -t test build` (7) · `nx run-many -t e2e` (2, excludes `visual-e2e`)
· `prettier --check .` · `tsc --noEmit` on `apps/visual-e2e/tsconfig.json`.
