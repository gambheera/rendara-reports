## Summary

Stands up **all four test types from day one** so every later story can include
tests immediately (brief §9). Vitest for unit + Angular component tests,
Playwright for e2e, and axe-core for accessibility — with **coverage thresholds
enforced by the test runner** (engine/schema ≥ 90%, UI ≥ 80%).

See [`docs/testing/test-harness.md`](../testing/test-harness.md) for the full guide.

### Key decisions

- **Runner: Vitest via the inferred `@nx/vitest/plugin`** (a `test` target on
  every lib/app), matching the inferred-plugin style already used for lint.
  Generated with `@nx/vitest:configuration`, then `convert-to-inferred` to avoid
  the v24-removed `@nx/vitest:test` executor. Deprecated `@nx/vite` path/asset
  plugins were replaced with `vite-tsconfig-paths` — the harness runs
  **warning-free**.
- **Angular component tests run zoneless.** `@analogjs/vite-plugin-angular`
  compiles components; `@analogjs/vitest-angular/setup-testbed` initialises a
  `provideZonelessChangeDetection()` TestBed. **No `zone.js`** is installed —
  the first-party `@angular/build:unit-test` builder was not used because it
  mandates a `buildTarget` the unbuilt Angular libs don't have.
- **Coverage enforced at the runner, not in CI.** Each config sets
  `coverage.enabled: true` + `coverage.thresholds`, so `nx test <project>`
  itself fails below the bar — independent of any pipeline. **E0-S3** then runs
  `nx affected -t lint test build e2e`; no threshold logic is duplicated there.
  (Approved as **option A** for the "enforced in CI" criterion while E0-S3 is
  pending.)
- **Playwright configs are self-contained `.mts`** that do **not** import
  `@nx/devkit`/`@nx/playwright/preset` — Playwright loads them through Node's
  ESM loader and pulling Nx's native addon in that path crashes. Chromium-only
  with a fixed `1280×720` viewport (deterministic; also seeds E0-S5's stable
  visual-regression environment). The inferred `e2e → <app>:serve` dependency is
  cleared so Playwright starts the dev server **once** (otherwise Nx's
  recursive-task guard aborts the run).
- **Shared axe helper** (`tools/testing/axe.ts`) asserts WCAG 2.2 AA; a narrow
  `tools/testing/**` carve-out was added to the `@nx/enforce-module-boundaries`
  `allow` list (it does not touch the §4 layer chain).
