## Summary

Adds **Storybook 10** to the three component libraries ŌĆö `ui-kit`,
`report-renderer`, `report-viewer` ŌĆö so components are documented and visually
testable in isolation, with the static build gated in CI. One example story per
lib renders against the current E0-S2 skeleton components; real components and
rendered output replace them from E0-S8 / Epic 4. Full guide:
[`docs/tooling/storybook.md`](../tooling/storybook.md).

### Key decisions (see [ADR 0002](../adr/0002-storybook-per-project-zoneless.md))

- **One Storybook per library** (`.storybook/` + `storybook`/`build-storybook`
  targets in each `project.json`), mapping directly onto "runs for X, Y, Z".
  Ports 4400 / 4401 / 4402; output `dist/storybook/<project>`.
- **Composition host** (root `.storybook/`, port 4500, `pnpm storybook:compose`)
  aggregates the three lib Storybooks under one sidebar via Storybook
  Composition `refs`, keeping them isolated. It renders no Angular, so it uses
  the lightweight `@storybook/html-vite` builder (the Angular dev-server builder
  can't run a refs-only host via the CLI). Dev-time only; not a CI gate.
- **Storybook 10.4.6** (`storybook`, `@storybook/angular`, `@storybook/addon-docs`).
  `@storybook/angular@10` is the line that supports **Angular 21** (`>=18 <22`);
  Storybook 9 caps at `<21`.
- **Zoneless via the builder.** `experimentalZoneless: true` bootstraps stories
  with the zoneless change detector; `zone.js` is an *optional* peer and is **not**
  installed. Proven by a `--smokeTest` render that boots the preview without Zone.js.
- **Dev-tooling only.** All four packages are `devDependencies`; the Angular
  Storybook builder's `@angular-devkit/build-angular` (webpack) is build-time
  only and never enters the `report-viewer` bundle ŌĆö the UI-kit-light rule holds.
- **Explicit targets, no `@nx/storybook` plugin.** `build-storybook` is cached in
  `nx.json`; `.storybook/**` + `*.stories.ts` are excluded from the `production`
  input, each `tsconfig.lib.json`, and vitest coverage, so stories never enter
  the publishable build or drag coverage.
