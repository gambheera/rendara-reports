# chore(workspace): initialize Nx workspace with Angular 21 toolchain (E0-S1)

> Story: **E0-S1 ¬∑ Initialize Nx workspace** (Epic 0 ‚Äî Foundations).
> Single-story PR. Base: `main` ‚Üí Head: `feat/e0-s1-init-nx-workspace`.

## Summary

Sets up the foundational Nx 23 monorepo and the shared toolchain that all later
Rendara stories build on. The workspace is intentionally **empty** ‚Äî no `apps/*`
or `libs/*` projects yet (those are scaffolded in **E0-S2**).

### Key decision: classic layout, not the TS-solution layout

Nx's empty/`apps` preset scaffolds the TypeScript **project-references ("TS
solution") layout**, which **`@nx/angular` does not support**
([Angular #37276](https://github.com/angular/angular/issues/37276)). Because
Angular is the foundation of the entire product, the workspace was bootstrapped
from the `angular-monorepo` preset to obtain the Angular-compatible **classic
layout** and a mutually-compatible version set, then reduced to a clean empty
workspace. The preset's demo app/libs and front-end-inappropriate dependencies
(express, `@nx/node`, `@nx/docker`, `@angular/ssr`, ‚Ä¶) were excluded. Angular
project generation was verified with `nx g @nx/angular:library --dry-run`
(writes nothing).

## What's included

- **Nx 23** (classic/integrated layout).
- **pnpm** package manager (pinned via `packageManager` + `.nvmrc`).
- **Angular 21** toolchain (`@nx/angular` 23, `@angular/*` 21.2.x); standalone +
  zoneless conventions; generator defaults (`prefix: rdr`, `style: css`).
- **Strict TypeScript** in `tsconfig.base.json`.
- **ESLint** flat config + **angular-eslint** (`21.3.1`) +
  `@nx/enforce-module-boundaries` (wired; real Rendara
  `schema ‚Üí engine ‚Üí renderer ‚Üí viewer` constraints land in E0-S2).
- **Prettier**, **EditorConfig**, **.gitattributes** (LF normalization),
  **.gitignore**.
- Root **README** placeholder documenting the workspace commands and a `smoke`
  script.
