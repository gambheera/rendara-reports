# docs: repo governance docs (E0-S9)

> Story: **E0-S9 · Repo governance docs** (Epic 0 — Foundations).
> Single-story PR. Base: `main` → Head: `feat/e0-s9-governance-docs`.

## Summary

Writes down the rules that the brief + `CLAUDE.md` already imply, and adds the
GitHub governance scaffolding the earlier Epic 0 ADRs deferred to this story. This
is a **docs-only / governance** story — no app, lib, schema, or renderer code is
touched.

Adds the five artifacts the story asks for:

- [`ARCHITECTURE.md`](../../ARCHITECTURE.md) — a navigable architecture **map**
  (two-part product, monorepo layout, dependency rules, the core pieces, the hard
  rules, tooling/quality, build sequence). It **links** the authoritative sources
  (brief, ADRs, deeper docs) rather than duplicating them.
- [`CONTRIBUTING.md`](../../CONTRIBUTING.md) — how we work: one-story-at-a-time +
  plan-first, branching/Conventional Commits, the DoD, quality-gate commands, Nx
  module boundaries, the Changesets rule, the ADR process, and the hard rules.
- [`docs/adr/template.md`](../adr/template.md) — the formal ADR template, matching
  the shape of the existing ADRs 0001–0004.
- [`docs/adr/0005-stack-and-architecture-decisions.md`](../adr/0005-stack-and-architecture-decisions.md)
  — the **foundational** ADR recording the brief §3–§4 stack/architecture
  decisions (Nx, Angular 21 zoneless, JSONata sandbox, single DOM renderer, custom
  paginator, native print + pluggable PDF, the boundary layering, etc.).
- [`.github/PULL_REQUEST_TEMPLATE.md`](../../.github/PULL_REQUEST_TEMPLATE.md) —
  auto-populated PR template embedding the **full DoD checklist (brief §9)**.
- [`.github/CODEOWNERS`](../../.github/CODEOWNERS) — default owner `@gambheera`
  plus path entries documenting the high-sensitivity areas (schema contract,
  viewer public API, contract docs, ADRs, CI).

Plus small supporting changes: [`docs/adr/README.md`](../adr/README.md) (ADR index
+ numbering convention) and a **"New here?"** pointer in the [README](../../README.md).

### Numbering note (the "first ADR")

The backlog calls this the "first ADR (stack decisions)," but ADRs 0001–0004 were
already written in E0-S5…S8 (ahead of the template, as each of them notes). ADRs
are **append-only / numbered by creation order**, so the foundational stack ADR is
**0005**, framed as the baseline the others refine. Renumbering existing ADRs was
rejected — it would break the cross-links in the README, prior PR docs, and the
ADRs themselves.
