# chore(tooling): add release tooling & commit hygiene (E0-S7)

> Story: **E0-S7 · Release tooling & commit hygiene** (Epic 0 — Foundations).
> Single-story PR. Head: `feat/e0-s7-release-tooling` (rebase onto `main` once
> earlier E0 stories — incl. **E0-S3 (CI)** — land).

## Summary

Wires **versioned releases** and **commit hygiene** before any feature code:
**Changesets** for the two publishable packages, **commitlint + Conventional
Commits** enforced by a **husky** `commit-msg` hook and a CI job, CHANGELOG
generation, and a **publish dry-run** (no real publish — packaging lands in E9).
Full guide: [`docs/tooling/releases.md`](../tooling/releases.md).

### Key decisions (see [ADR 0003](../adr/0003-release-tooling-changesets-commitlint.md))

- **Two publishable packages registered in the pnpm workspace.** Changesets
  discovers packages via `pnpm-workspace.yaml` `packages:`, which was previously
  empty by design. Only `libs/report-schema` and `libs/report-viewer` are added
  there, each with a **minimal `package.json`** (`@rendara/report-schema` /
  `@rendara/report-viewer`, `version: 0.0.0`, `publishConfig.access: public`).
  Other libs/apps stay pure-Nx and invisible to Changesets. Internal dev linking
  is unchanged (TS path aliases); `nx show projects` confirms project names are
  unaffected (`project.json` name still wins).
- **No package build yet.** These manifests carry no `exports`/`files`/build —
  ng-packagr/APF packaging and real `changeset publish` are **E9**. Until then
  `pnpm -r publish --dry-run` is the wired release check ("no real publish yet").
- **commitlint `config-conventional`**, with `body-/footer-leading-blank` left as
  warnings so the required `Co-Authored-By:` trailer never hard-fails. Existing
  history (`chore(tooling)`, `docs(test)`, …) passes unchanged.
- **husky v9** installs the hook on `pnpm install` (`prepare: husky`); the
  generated `.husky/_/` is git-ignored and regenerated per clone — only
  `.husky/commit-msg` is tracked.
- **CI re-checks** commit messages over the whole PR range (local hooks can be
  `--no-verify`'d). Like `storybook.yml`, `release.yml` is a thin stub for **E0-S3**
  to fold into the main `nx affected` pipeline.

## Story-specific QA

- **A non-conventional commit is rejected** — end-to-end via the husky hook:
  ```
  $ git commit --allow-empty -m "bad message no type"
  ✖ subject may not be empty [subject-empty]
  ✖ type may not be empty [type-empty]
  husky - commit-msg script failed (code 1)        # commit aborted, none created
  ```
  A conventional message (`chore(tooling): … (E0-S7)`), including the
  `Co-Authored-By:` trailer, passes (exit 0).

- **`changeset version` dry-run produces a correct bump** — with a temporary
  changeset (minor, both packages):
  ```
  $ pnpm release:status
  info Packages to be bumped at minor
    - @rendara/report-schema  0.1.0
    - @rendara/report-viewer  0.1.0
  $ pnpm release:version      # bumps both 0.0.0 → 0.1.0, writes CHANGELOG.md
  ```
  The QA changeset + generated CHANGELOGs were **reverted**; versions ship at
  `0.0.0` and no consumable changeset is committed (Epic 1+ PRs add real ones).

- **Publish dry-run:**
  ```
  $ pnpm release:dry-run
  📦 @rendara/report-schema@0.0.0 → https://registry.npmjs.org/
  [WARN] Skip publishing @rendara/report-schema@0.0.0 (dry run)
  📦 @rendara/report-viewer@0.0.0 → …  [WARN] Skip publishing … (dry run)
  ```

Other gates stay green: `nx run-many -t lint` (10 projects) · `nx run-many -t
test -p report-schema report-viewer` (100% cov) · `prettier --check .` ·
`nx show projects` (10, names unaffected by the new manifests).
