# chore(workspace): scaffold projects and enforce module boundaries (E0-S2)

> Story: **E0-S2 · Project skeleton & module boundaries** (Epic 0 — Foundations).
> Single-story PR. Base: `feat/e0-s1-init-nx-workspace` → Head:
> `feat/e0-s2-project-skeleton-module-boundaries` (stacked on E0-S1, not yet
> merged to `main`).

## Summary

Scaffolds the seven projects of the Rendara monorepo and replaces E0-S1's
placeholder `@nx/enforce-module-boundaries` config with the real **brief §4**
dependency rules, encoded as Nx `scope:*` / `type:*` tags. Every project is an
intentional **skeleton** (one placeholder export each) — real feature code
lands in later epics. The layers are now structurally unable to bleed into each
other.

## What's included

- **Seven projects** (per brief §4):
  | Project | Generator | `scope` / `type` tags |
  | --- | --- | --- |
  | `libs/report-schema` | `@nx/js:library` | `scope:schema` · `type:util` |
  | `libs/report-engine` | `@nx/js:library` | `scope:engine` · `type:util` |
  | `libs/report-renderer` | `@nx/angular:library` | `scope:renderer` · `type:feature` |
  | `libs/report-viewer` | `@nx/angular:library` | `scope:viewer` · `type:publishable` |

- TS path aliases `@rendara/*` (auto-added to `tsconfig.base.json`).

## Reconciliation / scope decisions

- **Inferred lint, not the deprecated executor.** E0-S1 flagged that
  `@nx/angular` emits a deprecated `@nx/eslint:lint` **executor** target. The
  explicit `lint` targets were stripped from every generated `project.json`, so
  all seven projects use the inferred `@nx/eslint/plugin` (consistent with the
  `@nx/js` libs). Resolves E0-S1's "Notes for E0-S2".
- **Not publishable yet.** `--publishable` pulls in Verdaccio + an Nx release
  config — that is **E0-S7** (release tooling) and **E9** (packaging). Libs are
  generated non-publishable; the correct `@rendara/report-schema` /
  `@rendara/report-viewer` import paths are auto-derived now regardless.
- **Test runners deferred to E0-S4.** Projects are generated with
  `unitTestRunner=none` / `e2eTestRunner=none`; the persisted generator
  defaults were reverted in `nx.json` so they stay neutral for E0-S4.
- **No Nx welcome page.** The generated `nx-welcome.ts` marketing component was
  removed from both apps in favor of a minimal skeleton shell.

## Story-specific QA — illegal import fails, legal passes

- **Legal imports pass (CI-enforced):** `nx run-many -t lint` → **green for 7
  projects** (the wired schema→engine→renderer→viewer chain + app edges).
- **Illegal import fails lint (verified, then reverted):**
  - `ui-kit` → `report-schema` (non-circular) →
    `error  A project tagged with "scope:ui-kit" can only depend on libs tagged with "scope:ui-kit"  @nx/enforce-module-boundaries`
  - `report-schema` → `report-renderer` (the story's example; closes a cycle) →
    `error  Circular dependency between "report-schema" and "report-renderer" detected: report-schema -> report-renderer -> report-engine -> report-schema  @nx/enforce-module-boundaries`
  - Procedure documented in
    [`docs/architecture/module-boundaries.md`](../architecture/module-boundaries.md).
